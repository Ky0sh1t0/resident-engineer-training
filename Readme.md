# Resident Engineer Training - C# Console System

Сборник задач над которыми я работал с прогрессией, при помощи C# и .NET

## Стэк технологий

- **Язык:** C#
- **Платформа:** .NET
- **Тип приложений:** Console Application

**Подход и концепция:**
- ООП (классы, объекты, свойства, методы)
- Работа с коллекциями(`List<type>`)
- Валидация ввода
- Обработка ошибок
- Управление состоянием объектов
- Архитектура консольных приложений

## Реализованные кейсы

### Case 01 - Access Control 

Базовое консольное приложение для ввода и отображения данных пользователя

**Реализовано:**
- Ввод имени, возраста, любимой игры и сколько часов проводит в неделю за играми
- Преобразования строки в число
- Вывод данных

**Освоенные навыки:** Переменные, Условные операторы (if, else), Базовая логика

### Case 06 - Persistance & Incident System

Расширенная версия Case 05 с сохранением данных о `Subjects`

**Реализовано:**
- Сохранение состояния данных в файле `subjects.json`
- Загрузка сохранённых данных
- Добавление случайного ивента в виде "побег" субъекта
- Улучшение архитектуры с помощью `enum`

**Добавленные механики:**
- 10% шанс случайного побега субъекта после действия пользователя, а именно при смене статуса и добавлении субъекта
- Запись данных при выходе и их загрузка при запуске

**Освоенные навыки:**
- Сериализация и десериализация `JSON`
- Использование `enum` для моделирования состояния
- Работа с файлами

**Пример сохранённого JSON**
```json
[
  {
    "Name": "Subject A",
    "DangerLevel": 5,
    "IsInfected": true,
    "Status": 1
  },
  {
    "Name": "Subject B",
    "DangerLevel": 4,
    "IsInfected": true,
    "Status": 1
  },
  {
    "Name": "Subject C",
    "DangerLevel": 2,
    "IsInfected": true,
    "Status": 0
  }
]
```

### Case 07 — Multi-File Project Architecture

Рефакторинг проекта с переходом от однофайловой структуры к многофайловой архитектуре.

**Реализовано:**
- Разделение проекта на файлы и папки (`Models`, `Enums`, `Services`)
- Вынос логики приложения в `SubjectService`
- Выделение модели `Subject` в отдельный файл
- Выделение enum-типов (`SubjectStatus`, `DisplayMode`)
- Упрощение `Program.cs`
- Сохранение полной работоспособности проекта после рефакторинга

**Освоенные навыки:**
- организация структуры C# проекта
- работа с `namespace`
- разделение ответственности между слоями
- рефакторинг без потери функциональности
